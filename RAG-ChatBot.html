Course Materials RAG System

RAG-Chatbot 项目介绍

一个面向课程材料的检索增强生成应用：把课程文稿切分、向量化、持久化， 再通过网页聊天界面回答用户关于课程内容的问题。

后端 FastAPI、ChromaDB、sentence-transformers、Zhipu/Z.ai GLM
前端 原生 HTML、CSS、JavaScript，无构建工具
语料 docs/ 中的课程文稿作为 RAG corpus
入口 backend/app.py 提供 API 与静态页面服务

项目定位

该项目是一个“课程材料问答”RAG 应用。它让用户不需要手动翻阅课程文稿， 就能直接询问某门课、某节课或某个主题的内容。系统会先检索相关课程片段， 再调用大模型生成简洁、上下文相关的回答。

它不是通用聊天机器人，而是围绕课程资料构建的专用课程助手。回答中会带上 sources，帮助用户知道答案来自哪门课程或哪一节课。

后端模块

  • backend/app.py：FastAPI 入口，定义 API、CORS、启动加载和静态文件服务。
  • backend/rag_system.py：RAG 编排层，连接文档处理、向量库、模型生成和会话管理。
  • backend/document_processor.py：读取课程文稿，解析课程和 lesson 元数据，并切分文本。
  • backend/vector_store.py：使用 ChromaDB 存储课程目录和课程内容向量。
  • backend/ai_generator.py：调用 Zhipu/Z.ai GLM，并处理 function calling。
  • backend/search_tools.py：定义 search_course_content 工具，供模型检索课程内容。
  • backend/session_manager.py：维护内存会话历史。
  • backend/models.py：定义 Course、Lesson、CourseChunk 数据模型。

核心数据流

1. 加载语料启动时读取 docs/ 下的课程文稿。
2. 文档切分解析课程、lesson，并按句子生成 chunk。
3. 向量存储课程元数据和内容分别写入 ChromaDB collection。
4. 用户提问前端调用 POST /api/query。
5. 检索生成模型调用搜索工具，结合结果生成最终回答。

查询时，系统会把问题包装成课程问答 prompt。模型如果判断需要课程材料，会调用 search_course_content，该工具在 ChromaDB 中做语义搜索，并把相关内容返回给模型。

向量检索设计

项目使用两个 ChromaDB collection：

  • course_catalog：存课程标题、讲师、课程链接和 lesson 列表，用于课程名模糊匹配。
  • course_content：存课程内容 chunk，用于实际语义检索。

这种拆分让用户可以输入不完整课程名，例如 “MCP 那门课”，系统先解析最接近的课程， 再在对应课程内容中检索。

API 接口

POST /api/query

用于课程问答，请求包含用户问题和可选 session。

{
  "query": "What does the MCP course say about tools?",
  "session_id": "session_1"
}

响应包含答案、来源和会话 ID。

{
  "answer": "...",
  "sources": ["MCP: Build Rich-Context AI Apps with Anthropic - Lesson 3"],
  "session_id": "session_1"
}

GET /api/courses

返回课程数量和课程标题列表。

{
  "total_courses": 4,
  "course_titles": ["...", "..."]
}

前端功能

  • 通过聊天框发送课程问题。
  • 支持回车发送和 suggested questions。
  • 助手回答支持 Markdown 渲染。
  • 显示课程数量和课程标题。
  • 回答下方可折叠显示 sources。

配置与运行

1. 进入项目目录

cd D:\Projects\CodexProjects\RAG-Chatbot

2. 准备 .env 配置

如果还没有 .env，先从模板复制：

Copy-Item .env.example .env

然后在项目根目录的 .env 中填写智谱 / Z.ai API Key：

ZAI_API_KEY=your_zhipu_api_key_here
ZAI_MODEL=glm-4.7-flash

3. 安装依赖

本机如果没有直接可用的 uv 命令，可以通过 python -m uv 调用：

python -m uv sync --dev

4. 首次缓存 embedding 模型

项目使用 sentence-transformers/all-MiniLM-L6-v2 生成向量，首次运行前需要下载并缓存一次：

.\.venv\Scripts\python.exe -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('all-MiniLM-L6-v2'); print('embedding model cached')"

5. 启动服务

推荐进入 backend/ 后启动，这与项目里的静态文件和 docs/ 路径设计一致：

cd backend
$env:HF_HUB_OFFLINE='1'
$env:TRANSFORMERS_OFFLINE='1'
..\.venv\Scripts\python.exe -m uvicorn app:app --host 127.0.0.1 --port 8000

后续再次运行时，可以直接重复上面的启动命令。

6. 访问应用

http://127.0.0.1:8000
http://127.0.0.1:8000/docs
http://127.0.0.1:8000/api/courses

7. 运行测试

python -m uv run pytest

当前特点与限制

特点

  • RAG 各层职责清晰，便于理解和二次开发。
  • 支持课程名模糊匹配和 lesson 过滤。
  • ChromaDB 持久化向量数据，避免每次启动都重新构建。
  • 前后端结构轻量，适合教学项目和原型验证。

限制

  • 会话历史只存在内存中，服务重启后会丢失。
  • 文档读取逻辑主要面向文本课程文稿。
  • 当前没有课程上传和重新索引 API。
  • CORS 和 TrustedHost 当前放开为 *，更适合本地开发。
  • 测试覆盖较少，主要覆盖共享 Pydantic 模型。

后续改进方向

  1. 增加课程上传、删除和重新索引接口。
  2. 把 session 存到 SQLite 或 Redis。
  3. 支持 PDF 和 DOCX 的真实解析。
  4. 在 sources 中显示课程链接、lesson 链接和命中文本片段。
  5. 增加检索质量评估测试。
  6. 限制生产环境下的 CORS、TrustedHost 和日志内容。

docs/ 中出现的 provider 或 model 名称是课程原始语料内容， 不代表应用本身依赖这些供应商。项目运行时使用的是 Zhipu/Z.ai 配置。
